---

## Ways to Contribute

- Report bugs
- Propose new features or improvements
- Improve UI/UX and accessibility
- Improve responsive layouts for mobile and web
- Improve documentation (README / examples / translations)
- Improve tests, architecture, and CI/CD workflows

---

## Before You Start

- Check existing issues and pull requests first.
- Keep changes focused and small.
- For behavior changes, include clear reproduction and verification steps.
- Follow the existing project architecture and code style.

---

## Project Stack

This project uses:

- Flutter
- Dart
- Material Design 3
- MVVM Architecture
- Provider
- GoRouter
- Easy Localization (i18n)
- Sentry
- Responsive Layout System

---

## Development Setup

1. Fork and clone the repository.
2. Follow the [Installation & Setup](README.md#installation--setup) steps in the README.
3. Create a branch:
   - `feat/<short-name>` for features
   - `fix/<short-name>` for bug fixes
   - `docs/<short-name>` for documentation
   - `refactor/<short-name>` for refactors
   - `chore/<short-name>` for maintenance tasks

4. Make your changes.
5. Run formatting, analysis, and tests before committing.
